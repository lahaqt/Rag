# 搜索对话功能设计

## 1. 背景与目标

当前前端左侧导航有一个「搜索对话」入口，但点击后仅展示占位页。本设计实现该功能：用户在独立搜索页输入关键词，系统按对话标题、摘要和消息内容做匹配，结果可点击跳转并滚动到命中消息。

## 2. 范围

- 仅实现**前端搜索**。
- 搜索范围：对话标题、摘要、消息内容。
- 匹配方式：大小写不敏感的子串匹配。
- 搜索结果点击后进入对应对话视图，并滚动到命中的消息。

不覆盖：后端搜索接口、数据库存储、知识库文件搜索、引用来源搜索、模糊/分词搜索。

## 3. 数据模型

复用前端现有数据结构：

- `Conversation`：`id`、`title`、`summary`、`time`、`messages`
- `Message`：`id`、`role`、`content`
- `messagesByConversation`：按 conversation id 分组的实际消息记录

## 4. 状态设计

在 `App.tsx` 中新增：

- `searchQuery: string` — 搜索框当前值。
- `scrollToMessageId: number | null` — 从搜索结果跳转过来时需要滚动到的消息 id，进入 chat 视图并滚动后清空。

## 5. 搜索逻辑

1. 对 `searchQuery.trim().toLowerCase()` 得到关键词。
2. 如果关键词为空，返回空结果或展示提示态。
3. 遍历 `conversations`：
   - 若 `title` 或 `summary` 包含关键词，标记该会话命中。
   - 遍历该会话的 `messagesByConversation[id]`，若某条消息 `content` 包含关键词，记录该消息 id 和简短片段。
4. 返回结果列表，每个结果包含会话信息和命中的消息片段数组。

## 6. UI/UX

### 6.1 搜索页布局

替换当前 `activeView === 'search'` 占位页：

- 顶部大搜索框（带 Search 图标、placeholder「搜索历史对话…」）。
- 输入为空时：显示提示文案，例如「输入关键词搜索历史会话」。
- 有结果时：按会话分组展示卡片，每张卡片显示：
  - 会话标题
  - 摘要（命中时高亮关键词）
  - 命中消息片段列表（每条显示角色标识和含高亮的简短内容）
- 无结果时：显示「未找到相关对话」。

### 6.2 结果交互

- 点击整个会话卡片：进入 chat 视图，`activeId` 设为该会话 id。
- 点击具体命中消息片段：进入 chat 视图，`activeId` 设为该会话 id，`scrollToMessageId` 设为该消息 id。
- 进入 chat 视图后，通过 `useEffect` 在消息列表渲染完成后滚动到 `scrollToMessageId` 对应的 DOM 元素，随后清空该状态。

### 6.3 高亮

对命中的标题、摘要、消息片段中的关键词用 `<mark>` 或样式包裹高亮。

## 7. 组件与代码位置

- 所有逻辑在 `frontend/src/App.tsx` 中实现，保持与当前项目一致的写法。
- 若搜索页 JSX 过大，可拆出 `ConversationSearch` 子组件，但第一阶段以内联为主。

## 8. 边界情况

- 搜索框为空：不显示结果，显示提示态。
- 关键词无命中：显示空态文案。
- 会话无消息（只有 title/summary）：仍可被标题/摘要命中。
- 跳转时目标消息不存在（数据不同步）：静默失败，仅进入对话视图。

## 9. 后续可扩展

- 后端持久化后替换为 `/api/conversations/search`。
- 增加模糊搜索、拼音匹配、按时间过滤。
- 支持搜索结果中的消息直接预览上下文。
